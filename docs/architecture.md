# Bayesian-AGI-Core 架构文档

## 1. 系统概述

Bayesian-AGI-Core 是一个基于自由能原理和主动推理的认知智能体项目，采用微服务架构，包含多个服务组件：API Gateway、LLM Service、Memory Service、Cognition Service、Vision Service、Multimodal Service。

### 2.2 服务组件

| 服务名称 | 端口 | 功能描述 |
|---------|------|----------|
| API Gateway | 8000 | 处理所有HTTP请求并路由到相应的微服务 |
| LLM Service | 8001 | 处理与LLM相关的请求，如获取模型列表 |
| Memory Service | 8002 | 处理与记忆相关的请求，如添加记忆、检索记忆 |
| Cognition Service | 8003 | 处理与认知相关的请求，如做出决策 |
| Vision Service | 8004 | 处理与视觉相关的请求，如图像分类、目标检测 |
| Multimodal Service | 8005 | 处理多模态输入，如文本、图像、音频 |

### 2.3 数据流程

1. 客户端发送请求到API Gateway
2. API Gateway根据请求类型路由到相应的微服务
3. 微服务处理请求，可能需要访问其他服务或存储
4. 微服务返回结果给API Gateway
5. API Gateway返回结果给客户端

## 3. 核心功能模块

### 3.1 记忆系统

记忆系统采用分层存储策略：
- 短期记忆：内存缓存（最近访问的记忆）
- 中期记忆：向量存储（ChromaDB）
- 长期记忆：关系型数据库（PostgreSQL）

记忆系统还包括知识图谱，用于存储和管理实体之间的关系。

### 3.2 认知系统

认知系统基于自由能原理和主动推理，包括：
- 贝叶斯大脑：处理不确定性和概率推理
- 主动推理：基于预测误差最小化的决策过程
- 双系统认知：系统1（快速、直觉）和系统2（慢速、分析）

### 3.3 多模态处理

多模态处理支持多种输入类型：
- 文本：处理文本输入，如总结、问答
- 图像：处理图像输入，如分类、检测、描述
- 音频：处理音频输入，如转录、分析

## 5. 部署架构

项目采用Docker Compose进行部署，包含以下服务：
- API Gateway
- LLM Service
- Memory Service
- Cognition Service
- Vision Service
- Multimodal Service
- RabbitMQ
- PostgreSQL
- Prometheus
- Grafana

## 6. 安全考虑

- CORS配置：限制来源域名
- 认证授权：预留接口，可集成JWT等认证机制
- 输入验证：对所有输入进行验证
- HTTPS：生产环境应使用HTTPS加密通信

## 7. 扩展点

- 插件系统：支持通过插件扩展功能
- 模型集成：支持集成不同的LLM模型
- 存储扩展：支持扩展到其他存储系统
- 服务扩展：支持添加新的微服务
